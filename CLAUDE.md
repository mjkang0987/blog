# CLAUDE.md — pikaworks blog 작업 규칙

이 저장소에서 블로그 글을 작성·수정할 때 지켜야 할 규칙입니다.
(발행 절차는 `README.md`, 편집 로그·소재 이력은 `plan.md` 참고)

## 표(table)로 수치를 정리한다

- **수치를 나열할 때, 표로 만들면 가독성이 나아지는 부분은 마크다운 표(GFM)로 작성한다.** 글머리표(`-`)로 수치를 여러 줄 늘어놓는 대신 표를 우선 검토한다.
- 표가 유리한 대표 사례:
  - 제품/모델별 사양·가격 비교 (같은 축으로 3개 이상 항목을 대조)
  - 여러 지표의 수치 대조, 전/후(before·after) 비교, 단계·일정별 수치
  - 2개 이상 열로 정렬되는 수치 묶음
- **역할 분담**: "왜 중요한가"라는 맥락은 문장으로 서술하고, 스캔용 수치는 표로 뺀다. 표와 본문이 같은 숫자를 통째로 중복하지 않게 한다.
- **모바일 가독성**: 열은 4개 이내 권장, 셀 내용은 간결하게. 공개되지 않은 항목은 억지로 채우지 말고 표 아래 한 줄로 밝힌다.
- **다국어 동기화**: 같은 글의 ko/en/ja 세 판에 표를 동일하게 반영한다.
- 표는 kramdown이 그대로 렌더링하며, 테마 CSS(`assets/css/main.css`의 `.post-content table`)에 헤더 배경·테두리·패딩 스타일이 이미 있다.

## 발행 게이트 (표를 넣어도 동일)

```bash
# 구조 검증(발행 차단 게이트, CI와 동일) — 표 줄(| … |)은 자동 스킵됨
python3 scripts/lint_structure.py _posts/<파일>.md --strict
# 한글판 윤문 점검(번역투·이중피동·어미 모노톤)
python3 scripts/humanize_ko.py --check _posts/<파일>.md
```

- 표 행이 길어 `humanize_ko.py`에서 '120자 장문' 경고가 날 수 있으나 비차단이며 허용된다(표는 산문 문장이 아님).
- OG 이미지는 `scripts/og_image.py --title "…" --out assets/og/<slug>-<lang>.png --category <카테고리> --theme dark` 로 생성한다.
